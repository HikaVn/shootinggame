# Audio Assets

This folder contains MP3 audio used by the game.

## Current files

- `bgm/*.mp3`: local placeholder BGM generated by `npm run gen-audio`.
- `sfx/*.mp3`: local synthesized sound effects generated by `npm run gen-audio`.
- `manifest.json`: maps logical game sound names to files and loop settings.

No third-party samples are used in the generated placeholder files.
Stage BGM files (`stage1`, `stage2`, `stage3`) are required to be at least 3 minutes long.

## Suno replacement workflow

Generate instrumental tracks in Suno, download them as MP3, then replace these files without changing code:

- `bgm/title.mp3`: title screen, not looped.
- `bgm/stage1.mp3`: Stage 1, looped.
- `bgm/stage2.mp3`: Stage 2, looped.
- `bgm/stage3.mp3`: Stage 3, looped.
- `bgm/boss.mp3`: boss battle, looped.

For looped tracks, export or edit the file so the opening and ending bars connect cleanly. If a Suno stage download is shorter than 3 minutes, the importer loops it to 185 seconds during import.

### Import command

Place the Suno downloads in one folder. The filenames must include these keys:

- `title`
- `stage1`
- `stage2`
- `stage3`
- `boss`

Then run:

```bash
npm run import-suno-bgm -- --src ~/Downloads/suno-bgm
```

To import only some tracks, pass a comma-separated target list:

```bash
npm run import-suno-bgm -- --src ~/Downloads --targets title,stage1,stage2,stage3
```

The importer normalizes loudness, converts to 192 kbps MP3, writes to `bgm/*.mp3`, extends `stage1`/`stage2`/`stage3` to at least 185 seconds when needed, and backs up the previous files under `bgm/_previous/`.

Verify the result with:

```bash
npm run verify-suno-audio
```

`npm run verify-audio` accepts the local placeholder BGM. `npm run verify-suno-audio` requires the manifest to show that Suno BGM has been imported. Both checks fail if any stage BGM is under 180 seconds.

Loop behavior is controlled by `manifest.json`, not by the filenames:

- `title`: no loop
- `stage1`, `stage2`, `stage3`, `boss`: loop
